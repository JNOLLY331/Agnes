05 — REST API

API Endpoints

All endpoints return JSON. Protected routes require an Authorization: Bearer <token> header. Admin routes additionally require is_staff=True.

Authentication
POST /api/users/register/ Register new user account
POST /api/users/login/ Returns access + refresh JWT tokens
POST /api/token/refresh/ Get new access token using refresh token
Users (Protected)
GET /api/users/profile/ Get current user's profile
PUT /api/users/profile/ Update name, email, password
GET /api/users/ List all users (Admin only)
Products
GET /api/products/ List products (supports ?keyword=&category=&page=)
GET /api/products/{id}/ Get single product detail
POST /api/products/ Create product (Admin only)
PUT /api/products/{id}/ Update product (Admin only)
DELETE /api/products/{id}/ Delete product (Admin only)
POST /api/products/{id}/reviews/ Submit a product review (Protected)
Orders (Protected)
POST /api/orders/ Create a new order from cart
GET /api/orders/myorders/ Get authenticated user's orders
GET /api/orders/{id}/ Get order detail
PUT /api/orders/{id}/pay/ Update order to paid (after payment)
PUT /api/orders/{id}/deliver/ Mark order delivered (Admin only)

Example: Protected Axios Request

// src/utils/axiosInstance.js import axios from 'axios' const api = axios.create({ baseURL: 'http://localhost:8000/api', }) // Attach JWT to every request automatically api.interceptors.request.use(config => { const token = localStorage.getItem('accessToken') if (token) config.headers.Authorization = `Bearer ${token}` return config }) export default api

07 — Development Process

10-Step Build Workflow

Follow this order to avoid having to redo work. Database first, then API, then test the API, then build the UI on top of working data.

Design the Database Schema

Draw your tables and relationships before writing any code. Identify primary keys, foreign keys, and which fields are nullable. This prevents costly migrations later.

PostgreSQLERD diagram

Set Up Django + DRF

Create virtual environment, install packages, configure settings.py (database, CORS, installed apps), write models.py and run migrations.

Djangomodels.pymigrations

Build the REST API

Write serializers, viewsets, and URL routes for all resources. Add JWT authentication. Use Django's browsable API to verify endpoints work.

DRFserializersviewsurls

Test APIs with Postman

Create a Postman collection. Test every endpoint — register user, login, list products, create order. Fix bugs before the frontend ever touches the API.

PostmanJWT testing

Scaffold React App

Create React app with Vite or CRA, set up React Router routes, install Tailwind, create folder structure: pages/, components/, redux/, utils/.

ReactViteTailwindfolder structure

Implement Authentication

Build login/register forms with Formik + Yup. Store JWT in localStorage. Create a Redux auth slice. Add protected route wrappers using React Router.

FormikYupReduxlocalStorage

Build Product Pages

Product listing with grid/list view, search bar, category filter, and pagination. Product detail page with image gallery, add-to-cart button, and reviews.

AxiosReact statepagination

Build Cart & Checkout

Cart stored in Redux (or localStorage). Checkout form for shipping address. Order summary page before payment. Post order to /api/orders/ on confirm.

Redux cartcheckout floworder API

Integrate Payments

Wrap checkout in Stripe Elements. On payment success, call /api/orders/{id}/pay/ to update order status. Handle M-Pesa STK Push callback from Daraja API.

StripeM-Pesa Darajawebhooks

Deploy Frontend & Backend

Push React to Vercel. Deploy Django to Render with gunicorn + whitenoise. Provision managed PostgreSQL. Set environment variables. Configure CORS for prod domain.

09 — Best Practices

Tips for Success

Hard-learned lessons from real e-commerce projects. These will save you hours of debugging.

Never store passwords in plain text

Django hashes passwords automatically with set_password(). Never store raw strings in the database. Use AbstractUser for custom user models.

Keep JWT tokens short-lived

Set access tokens to 5–15 minutes, refresh tokens to 7–30 days. Use the refresh token to silently get new access tokens — users stay logged in without security risk.

Validate stock before checkout

Always re-check product.stock server-side when an order is placed. Never trust the client. Use select_for_update() to prevent race conditions.

Use pagination everywhere

Add PageNumberPagination to your DRF settings. Returning all 10,000 products in one request will crash your app in production.

Handle CORS correctly

Add corsheaders to MIDDLEWARE before Django's common middleware. In development allow all origins, in production whitelist only your React domain.

Use Stripe webhooks for payments

Don't only rely on frontend callbacks to mark orders paid. Set up a /webhook/stripe/ endpoint that listens to payment_intent.succeeded events from Stripe's servers.

Store images on cloud storage

In production, save uploaded product images to AWS S3 or Cloudinary using django-storages. Never store user uploads on the server filesystem.

Write tests for your API

Use Django's APITestCase to test each endpoint. At minimum: can a user register, login, browse products, and place an order? Run tests before every deployment.

Use .env files everywhere

Never hardcode secrets in your codebase. Use python-decouple in Django and Vite's import.meta.env in React. Add .env to .gitignore immediately.
